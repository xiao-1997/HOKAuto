### 1.2 架构分层说明

| 层级         | 组件                    | 职责                                                |
| ------------ | ----------------------- | --------------------------------------------------- |
| **前端层**   | Vue + Ant Design        | 提供Web管理界面，包括设备管理、任务管理、监控面板等 |
| **服务端层** | Spring Boot / Node.js   | 设备注册与管理、任务调度、结果存储、API服务         |
| **Agent层**  | Python + Appium         | 部署在控制机上，负责设备发现、任务执行、状态上报    |
| **设备层**   | iPhone + WebDriverAgent | 执行具体的自动化操作                                |

## 2. 技术选型

| 层级           | 技术选型                               | 说明                                        |
| -------------- | -------------------------------------- | ------------------------------------------- |
| **前端**       | Vue 3 + Ant Design Vue                 | 用户指定的前端框架                          |
| **前端构建**   | Vite                                   | 快速构建与热更新                            |
| **服务端**     | Spring Boot (Java) / Node.js + Express | 二选一，根据团队技术储备决定                |
| **数据库**     | MySQL + Redis                          | MySQL存储持久化数据，Redis做缓存和消息队列  |
| **设备自动化** | Appium 2.x + WebDriverAgent            | iOS真机自动化驱动                           |
| **设备连接**   | usbmuxd + iproxy                       | iOS设备USB通信与端口转发                    |
| **Agent开发**  | Python 3.11+                           | 设备控制脚本与调度逻辑                      |
| **通信协议**   | REST API + WebSocket                   | REST用于任务下发，WebSocket用于实时状态推送 |
| **部署**       | Docker + Docker Compose                | 容器化部署，便于扩展                        |

## 3. 核心模块详细设计

### 3.1 服务端模块 - 数据模型

**设备表 (device)**：

````sql
CREATE TABLE device (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    udid VARCHAR(64) UNIQUE NOT NULL COMMENT '设备唯一标识',
    device_name VARCHAR(100) COMMENT '设备名称',
    model VARCHAR(50) COMMENT '设备型号',
    ios_version VARCHAR(20) COMMENT '系统版本',
    status ENUM('online', 'offline', 'busy', 'idle') DEFAULT 'offline',
    agent_id VARCHAR(64) COMMENT '所属Agent ID',
    last_heartbeat DATETIME COMMENT '最后心跳时间',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

任务表 (task)：
CREATE TABLE task (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(200) NOT NULL COMMENT '任务名称',
    script_content TEXT NOT NULL COMMENT '自动化脚本内容',
    target_app VARCHAR(100) COMMENT '目标应用Bundle ID',
    timeout INT DEFAULT 300 COMMENT '超时时间(秒)',
    priority TINYINT DEFAULT 5 COMMENT '优先级 1-10',
    status ENUM('pending', 'queued', 'executing', 'completed', 'failed', 'cancelled') DEFAULT 'pending',
    device_id BIGINT COMMENT '执行设备ID',
    agent_id VARCHAR(64) COMMENT '执行Agent ID',
    submitted_by VARCHAR(64) COMMENT '提交人',
    started_at DATETIME COMMENT '开始执行时间',
    finished_at DATETIME COMMENT '完成时间',
    result TEXT COMMENT '执行结果摘要',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
3.2 多设备并发控制方案
每台设备独立映射端口：
  iPhone 1 (UDID: xxx) → iproxy 8101 → 设备8100端口
  iPhone 2 (UDID: yyy) → iproxy 8102 → 设备8100端口
  iPhone N (UDID: zzz) → iproxy 810N → 设备8100端口

每台设备对应独立Appium Server：
  Appium Server 1 → --port 4723 --wda-local-port 8101
  Appium Server 2 → --port 4724 --wda-local-port 8102
  Appium Server N → --port 472N --wda-local-port 810N
````

## 8. 2026-06-25 iOS 原生自动化迭代

### 8.1 视觉驱动架构
```
Swift App → Lua 主循环 (5s 轮询)
            ├─ findImage 模板匹配 (3s 超时)
            │   └─ 匹配取消/关闭/登录/暂不参与按钮
            ├─ → 未命中 → 截图 base64 → DeepSeek API
            │   └─ AI 返回按钮坐标 → 点击
            └─ → 全失败 → 盲点坐标兜底
```

### 8.2 触控方案
- 通过 `su mobile -c 'autotouch ...'` 调用（Substrate 注入）
- AutoTouch Lua 脚本：keepScreen 截图 → findImage 模板匹配
- DeepSeek API：文本 + 图片多模态分析
