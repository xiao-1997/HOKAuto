# 系统需求文档：Web管理后台 + iPhone分布式自动化调度平台

## 1. 项目概述

### 1.1 项目背景

在移动互联网业务运营中，存在大量需要在多台iPhone设备上执行跨应用自动化操作的场景（如内容分发、数据采集、自动化测试等）。当前面临以下痛点：

- **设备管理混乱**：多台iPhone设备分散管理，无法实时掌握设备在线状态、忙碌/空闲状态
- **任务派发低效**：任务依赖人工分配设备，无法自动匹配空闲设备
- **执行状态不可视**：无法实时追踪任务执行进度和结果
- **设备利用率低**：缺乏统一的调度机制，设备闲置与任务积压并存

### 1.2 项目目标

搭建一套 **Web管理后台 + 多台iPhone分布式自动化调度平台**，实现以下闭环流程：

> 管理员在Web后台提交任务 → 服务端自动筛选空闲iPhone派发任务 → iPhone执行跨应用自动化操作 → 执行完成回传状态 → 设备自动恢复空闲

### 1.3 核心价值

| 价值点     | 说明                                                |
| ---------- | --------------------------------------------------- |
| 统一管控   | 通过Web平台集中管理所有iPhone设备，实时查看设备状态 |
| 智能调度   | 任务自动匹配空闲设备，实现设备资源最大化利用        |
| 流程闭环   | 从任务创建到执行完成全流程可追踪、可回溯            |
| 跨应用能力 | 支持在iPhone上执行跨多个应用的自动化操作序列        |

## 3. 功能需求

### 3.1 设备管理模块

| 功能             | 描述                                                         | 优先级 |
| ---------------- | ------------------------------------------------------------ | ------ |
| 设备注册         | iPhone接入系统后自动注册，记录UDID、设备型号、系统版本等信息 | P0     |
| 设备列表         | 展示所有设备及其状态（在线/离线/忙碌/空闲），支持搜索和筛选  | P0     |
| 设备状态实时更新 | 设备在线状态、任务执行状态实时同步到后台                     | P0     |
| 设备分组         | 支持按用途/项目对设备进行分组管理                            | P2     |
| 设备维护         | 支持远程重启设备、清理应用数据等维护操作                     | P2     |

### 3.2 任务管理模块

| 功能     | 描述                                                                            | 优先级 |
| -------- | ------------------------------------------------------------------------------- | ------ |
| 创建任务 | 管理员通过Web表单创建自动化任务，配置执行参数（目标应用、操作步骤、超时时间等） | P0     |
| 任务列表 | 展示所有任务及其状态（待执行/执行中/已完成/失败），支持筛选和排序               | P0     |
| 任务详情 | 查看任务完整信息，包括执行设备、执行日志、截图/录屏等                           | P0     |
| 任务取消 | 支持取消待执行或执行中的任务                                                    | P1     |
| 任务重试 | 支持失败任务一键重试                                                            | P1     |
| 定时任务 | 支持配置定时触发的自动化任务                                                    | P2     |

### 3.3 调度引擎模块

| 功能         | 描述                                                       | 优先级 |
| ------------ | ---------------------------------------------------------- | ------ |
| 设备自动筛选 | 任务下发时自动筛选状态为“空闲”且符合条件的设备             | P0     |
| 任务队列     | 当所有设备繁忙时，任务进入等待队列，按先进先出或优先级调度 | P0     |
| 负载均衡     | 多设备场景下均衡分配任务，避免单设备过载                   | P1     |
| 超时处理     | 任务执行超时自动中断并标记失败                             | P0     |

### 3.4 执行引擎模块（iPhone端）

| 功能       | 描述                                               | 优先级 |
| ---------- | -------------------------------------------------- | ------ |
| 任务接收   | iPhone Agent接收服务端下发的任务指令               | P0     |
| 自动化执行 | 基于Appium + WebDriverAgent执行跨应用自动化操作    | P0     |
| 状态回传   | 实时回传任务执行进度和状态                         | P0     |
| 日志记录   | 记录执行过程中的详细日志、截图，任务结束后打包上传 | P0     |
| 异常恢复   | 任务执行异常时自动重启连接或重试                   | P1     |

### 3.5 结果与报告模块

| 功能         | 描述                                             | 优先级 |
| ------------ | ------------------------------------------------ | ------ |
| 执行结果查看 | 在Web后台查看任务执行结果（成功/失败）及详细信息 | P0     |
| 执行日志     | 查看任务执行全过程的详细日志                     | P0     |
| 截图/录屏    | 查看执行过程中的截图和录屏回放                   | P1     |
| 统计报表     | 设备利用率、任务成功率、执行时长等统计图表       | P2     |

## 6. 2026-06-18 迭代更新

### 6.1 商品管理
- 管理员可添加/编辑/下架商品（名称、面值、图标、目标App）
- 商品数据存储于 product 表
- 默认商品：王者荣耀475点券、王者荣耀60点券

### 6.2 订单与兑换系统
- 下单流程：商品列表 → 选择商品 → 设置购买数量/倍拍数量 → 生成兑换码
- 每个兑换码对应一个独立订单，初始状态为"未使用"
- 订单状态：未使用 → 处理中 → 已使用 / 失败 / 作废 / 拦截
- 兑换中心：独立公开页面 /redeem/:code，无需登录，展示订单状态和兑换规则

### 6.3 订单管理
- 订单记录：表格展示订单号/商品/卡密/提交人/单价总价/数量/状态/时间
- 兑换管理：按处理中/已使用切换查看
- 支持作废、拦截操作，拦截通过WebSocket通知Agent停止

### 6.4 账号管理
- ID管理：导入平台Apple ID账号（支持4种格式：账号--密码--API--安全码）
- ID日志：展示账号充值消耗记录（订单号/ID/价格凭证/商品编号/时间）
- 平台账号管理：Web登录用户列表

### 6.5 个人中心
- 查看基本信息，修改登录密码

## 7. 2026-06-25 视觉自动化迭代

### 7.1 动态坐标识别
- 废弃固定坐标，采用 AutoTouch findImage 模板匹配动态获取按钮坐标
- 每 5 秒轮询：截图 → 图像预处理 → 离线模板匹配
- 适配弹窗位置浮动、分辨率变化

### 7.2 findImage 超时与 DeepSeek 兜底
- findImage 单次匹配 3 秒超时
- 3 秒内无匹配结果 → 自动触发 DeepSeek AI 截图分析
- DeepSeek 返回按钮名称和坐标 → 本地执行点击
- 全失败 → 盲点兜底坐标

### 7.3 按钮模板库
- 取消按钮 (cancel)、关闭按钮 (close/close2/x)
- 暂不参与 (skip)、登录按钮 (login)
- 图片存储在 /var/mobile/Library/AutoTouch/Scripts/Images/
